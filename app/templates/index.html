Endpoints:

  • GET /health — liveness + DB connectivity
  • GET|POST /api/profiles — list / create profiles
  • GET /api/profiles/<id> — profile + its posts
  • POST /api/profiles/<id>/avatar — upload picture (multipart) → S3
  • GET /api/profiles/<id>/avatar — fetch picture
  • GET|POST /api/profiles/<id>/posts — list / create posts
  • GET|POST /api/items — simple item CRUD
  • GET /api/cpu?ms=250 — CPU load (for autoscaling demos)
